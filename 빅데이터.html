package ontime;
import org.apache.hadoop.io.*;
import org.apache.hadoop.mapreduce.*;
import java.io.IOException;
public class SumReducer extends Reducer<Text, DoubleWritable, Text, 
DoubleWritable>{
private DoubleWritable result = new DoubleWritable();
public void reduce(Text key, Iterable<DoubleWritable> values, Context 
context) throws IOException, InterruptedException{
//Iterable is list, so send out every time one
double sum = 0;
int count =0;
for(DoubleWritable value : values) {
sum += value.get();
count++;
}
result.set(sum/count);
context.write(key, result);
}
}

RDD의 특징
■
불변성(immutable)
-
불변적 데이터 구조 : 한번 생성되면 제자리(in-place)에서는 수정할 수 없다.

-

■
파티션화(partitioned)
-
모든 RDD 데이터는 파티션으로 분할되어 클러스터에 분배된다.
-
RDD 파티션과 물리적 파티션 간에 매핑이 존재
■
결함 허용성(fault tolerant)
-
노드에 문제가 생겨 오류가 발생하면 스파크는 이 파티션을 다른 노드에 새로 생성한다.
-
스파크는 각 RDD 생성에 대한 계보 정보(lineage information)를 저장하고, 이를 이용하
여 일부 혹은 전체 RDD를 복구한다.
■
강-타입(strongly typed)
-
RDD 클래스는 타입 파라미터를 가진다.
-
RDD는 사용자 정의 클래스를 포함해서 파이썬, 자바, 스칼라의 어떤 타입의 객체든 가질 
수 있다.
■
인메모리(in-momory)
-
RDD 클래스는 인메모리 클러스터 컴퓨팅이 가능한 API를 제공한다.
-
스파크는 RDD를 캐시하거나 메모리에 상주시킬 수 있다.
